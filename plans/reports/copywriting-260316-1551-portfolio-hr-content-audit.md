# Portfolio Content Audit — HR Attractiveness

**Date:** 2026-03-16 | **Target audience:** Recruiters, HR managers, hiring managers
**Framework:** AIDA + 4Us + Landing Page Checklist
**Verdict:** 6/10 — Functional but underperforms for HR conversion. Fixable.

---

## Section-by-Section Review

### 1. Hero Section (Homepage) — 5/10

**Current copy:**
> "Hi, I'm Kane Nguyen"
> "I build web applications with a focus on performance and user experience. Passionate about clean code, modern tooling, and shipping products people love."

**Issues:**
| Problem | Why it matters for HR |
|---------|----------------------|
| Generic opening "Hi, I'm" | Every dev portfolio says this — no differentiation |
| "Passionate about clean code" | Overused buzzword, HR sees this 100x/day |
| No specific outcome/result | HR wants to know: what VALUE do you bring? |
| No social proof above fold | Missing: company worked at, users served, tech shipped |
| Rotating titles dilute focus | "Software Engineer / Web Developer / Problem Solver" — pick ONE and own it |

**Suggested rewrite:**
> **Kane Nguyen**
> Software Engineer who ships fast.
> Built web apps serving 500+ users with React, Next.js & Node.js. 1 year shipping production code — currently open to new opportunities.
> [View Projects] [Download Resume]

**Why better:** Specific (500+ users), action-oriented (ships fast), CTA includes resume (HR wants this immediately).

---

### 2. About Preview (Homepage) — 6/10

**Current copy:**
> "I'm a Software Engineer with 1 year of experience building modern web applications. I thrive on turning ideas into polished products — from architecting scalable backends to crafting pixel-perfect UIs."

**Issues:**
- "1 year of experience" as the lead = positioning yourself as junior first. Lead with what you built, not tenure.
- "thrive on turning ideas into polished products" — vague. What ideas? What products?
- Stats cards (1+ Year, 5+ Projects, 10+ Tech) are weak — "5+ Projects" doesn't impress. Reframe.

**Suggested rewrite:**
> Built 5 production apps across full-stack — from real-time task management (500+ users) to e-commerce APIs handling 1K+ orders/month. I pick up new stacks fast and ship clean, tested code.

**Stats reframe:**
| Current | Better |
|---------|--------|
| 1+ Year Exp | 1yr shipping production code |
| 5+ Projects | 5 production apps |
| 10+ Technologies | 10+ tools in my stack |

---

### 3. Bio Section (About page) — 4/10 ← WEAKEST

**Current copy:**
> "I build web apps with React, Next.js, and TypeScript — 1 year in and still enjoying the steep parts of the learning curve. I leave work at work, do nothing productive in my free time, and find comfort in both the boring and the difficult."

**Critical issues:**
| Problem | HR Red Flag Level |
|---------|-------------------|
| "do nothing productive in my free time" | **HIGH** — HR may read as lazy/unmotivated |
| "still enjoying the steep parts" | Positions you as still learning, not delivering |
| "I leave work at work" | Can read as "I don't go above and beyond" |
| No mention of achievements | Nothing concrete to evaluate |
| No mention of what you're looking for | HR doesn't know what role to pitch you |

**Suggested rewrite:**
> I build web apps with React, Next.js, and TypeScript. In my first year as a Software Engineer, I've shipped features that improved user engagement by 20%, built REST APIs for e-commerce, and set up CI/CD pipelines that cut production bugs by 30%.
> I value clean boundaries between work and life — it keeps my code sharp and my thinking fresh. Currently looking for a full-stack or frontend role where I can grow with a strong engineering team.

**Why better:** Leads with results, reframes "leave work at work" positively, tells HR exactly what role to pitch.

---

### 4. Project Descriptions — 7/10

**Good:** Each project has role, team size, impact, tech stack. This is above average for dev portfolios.

**Issues:**
- "Personal branding & job search" as impact for Portfolio v2 — not impressive. Use "Showcases 5 production projects to hiring teams"
- Generic descriptions: "Full-stack task management application with real-time updates" — add the WHY and the RESULT
- Missing: What problems did you solve? What did you learn?

**Suggested improvements:**
| Current | Better |
|---------|--------|
| "Full-stack task management application with real-time updates" | "Built for a 3-person team to ship faster — real-time task sync via WebSocket, now used by 500+ people" |
| "RESTful API for an e-commerce platform" | "E-commerce backend processing 1K+ orders/month — JWT auth, Stripe payments, PostgreSQL" |
| "Personal branding & job search" | "Built in 2 weeks with Next.js 15 — my proof of work for hiring teams" |
